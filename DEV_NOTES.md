# Arduino FPV 履带机器人 - 开发问题与解决方案记录

## 1. 硬件引脚定义问题
**问题描述**：
在使用 A4950 电机驱动时，使用代码 `const int PIN = 5;` 无法控制电机。
**原因分析**：
Arduino Uno Q (基于 STM32/Zephyr) 的底层引脚映射与传统 AVR Uno 不同。纯数字 `5` 指向的是芯片内部 GPIO 索引，而非板载丝印 `D5`。
**解决方案**：
必须使用带 `D` 前缀的常量定义。
- ❌ 错误：`const int PIN = 5;`
- ✅ 正确：`const int PIN = D5;`

## 2. 电机对向安装导致的逻辑冲突
**问题描述**：
发送“前进”指令时，小车原地打转；发送“左转”时小车前后拉扯。
**原因分析**：
履带底盘左右电机通常是**镜像对称安装**（背对背）。给同样的电平信号（例如都给正电压），物理上它们会朝相反的方向旋转。
**解决方案**：
在 C++ 底层驱动中，对右轮逻辑进行**反向处理**。
- 左轮：`Speed > 0` -> 正转逻辑
- 右轮：`Speed > 0` -> 反转逻辑

## 3. Servo 库与 analogWrite 的资源冲突
**问题描述**：
当代码中引入 `<Servo.h>` 并 attach 舵机后，原本控制电机的 `analogWrite()` 失效，电机不动。
**原因分析**：
`Servo` 库为了生成精准的 50Hz PWM 信号，占用了单片机的**硬件定时器 (Timer)**。在 Arduino 架构中，该定时器往往也负责特定引脚（如 D9, D10）的硬件 PWM 生成。定时器被抢占后，`analogWrite` 无法输出波形。
**解决方案**：
放弃 `analogWrite`，改用**非阻塞式软件 PWM**。利用 `micros()` 获取系统时间，手动快速翻转引脚电平来模拟 PWM 波形。

## 4. 舵机抖动 (Jitter)
**问题描述**：
舵机在静止时不断抽搐、抖动。
**原因分析**：
1. **代码阻塞**：使用了 `delay()` 或 `for` 循环来生成电机 PWM，导致 CPU 无法及时响应 Servo 的信号维持中断。
2. **电源噪声**：直流电机启动时的电流冲击干扰了舵机供电。
**解决方案**：
1. **代码优化**：移除所有 `delay()` 和死循环，使用非阻塞逻辑（见解决方案3）。
2. **硬件滤波**（建议）：在舵机电源处并联电解电容。
